呼叫物件屬性與方法

當您呼叫伺服端方法時，此方法並不會傳回一個單一值，而是建立一個呼叫物件。此物件中包含呼叫程序的傳回與狀態資訊。下表列出呼叫物件的屬性。

屬性 描述 
id
 呼叫的唯一值，在第一次呼叫時產生。 
return_value
 如果方法有傳回值的話，則此屬性為其傳回值。 
data
 伺服端回應遠端 scripting 呼叫所傳回的原始資料，包含在 XML 標記中。相關的詳細資訊，請參閱檢查錯誤。 
status
 此數值代表方法呼叫目前的狀態。可能的數值如下︰

-1   失敗

0   完成

1   暫停

相關的詳細資訊，請參閱非同步呼叫遠端 Scripting 方法。

 
message
 關於呼叫的文字資訊。如果呼叫成功，則 message 的內容為「完成」。如果呼叫失敗，則 message 的內容可能是伺服端所傳回的錯誤文字，也可能是標準的錯誤訊息。相關的詳細資訊，請參閱檢查錯誤。 
callback
 呼叫中所指定的選擇性回呼函數的文字。 
error_callback
 呼叫所指定的選擇性錯誤回呼函數的文字。 
context
 隨著此呼叫傳遞的內容值。相關的詳細資訊，請參閱非同步呼叫遠端 Scripting 方法。

下表列出此呼叫物件所能呼叫的方法。

方法 描述
wait 停止目前正在執行的客戶端 script，直到指定呼叫物件的非同步遠端 scripting 呼叫完成為止。
cancel 停止以非同步方式執行的遠端 scripting 呼叫。

關於呼叫物件的詳細資訊，請參閱 同步呼叫遠端 Scripting 方法和非同步呼叫遠端 Scripting 方法。
